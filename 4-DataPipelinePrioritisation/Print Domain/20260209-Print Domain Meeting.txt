Summary
Print Domain Data Model Restructuring
Mazen and Kevin discussed the data model for the Print Domain, focusing on migrating current tables to a new architecture with three layers: Confirmed Database, Metrics Database, and Semantics Database. They clarified that the source data is not ingested directly into the two facts tables but goes through several intermediate layers. Kevin expressed concerns about query performance in the new data model, particularly in the semantic layer. They discussed the challenges of handling revenue data at different levels and the need for a third normal form model in the Confirmed layer. Mazen suggested revisiting the data model when the project kicks off, as it should be designed to minimize expectations for consumers and ensure seamless access for business users.

Print Data Model Consolidation Plan
Mazen and Kevin discussed the transformation of print data models, focusing on reducing the number of models for better maintainability. Kevin explained that around 30 models, including intermediary and target models, need to be analyzed and potentially consolidated. Mazen suggested targeting a 30% reduction in the number of models, aiming for a more compact and easily maintainable system. They agreed that the biggest modeling challenge would come from the fact Print Funnel, which currently uses a convoluted data structure with over 50 events in one table.

Dependency Migration and Orchestration Plans
Kevin and Mazen discussed the migration of dependencies and the creation of new semantic models. They agreed that the current orchestration process is a daily cron job, but they plan to move it to Airflow. Kevin explained that there is a requirement for a near real-time 2-hour refresh of certain data, which is currently handled by a separate database. They discussed the possibility of combining this with the Fact Print Audit process, which also needs a 2-hourly refresh. Kevin proposed an event-based system for the Fact Print Ordered data, while the Print Funnel does not require this.

Print Funnel Data Migration Planning
Mazen and Kevin discussed the migration of print funnel data, including data volume, governance policies, and the need for documentation. They explored the complexity of current DBT models and the potential for evolving metrics. Kevin expressed a need for assistance with data modeling and building new DBT models. They agreed that productionization would be handled by Canva. Mazen noted that the next step would be to write up individual scoping documents for each domain, with Kevin's target to complete the fact print order by the end of March.
